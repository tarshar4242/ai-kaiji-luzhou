## 3. 為什麼這樣做

這一段是最重要的，因為步驟之後都看得到，但「當初為什麼這樣選」最容易忘記。

- **為什麼做成單一個 `index.html` 檔，而不是用複雜的網站系統？**
  因為這樣最好維護也最好搬家。它不需要架伺服器、不需要資料庫，雙擊就能在瀏覽器看，寄給別人、換主機都很方便。對「不寫程式、但要常常改內容」的你最合適。

- **為什麼內容要收進 `sections` 這張清單？**
  把「內容」和「版面長相」分開。這樣你（或我幫你）改文字時，只動這張清單，不會不小心弄壞排版。要新增一段課程，就在清單裡多加一筆，網頁會自動長出來。

- **為什麼課程劇本選 5/28 公務版，而不是更早的 5/14 版？**
  因為你手上有兩套劇本：早期 5/14 是「驚艷體驗」走向（畫圖、營養師、公仔、抽獎），5/28 是「公務現場實用」走向（四個監理所日常會用到的 Gem）。兩套混在一起會讓課程失焦，你選了貼近學員工作的公務版，所以網頁就以它為主軸。

- **為什麼放 GitHub Pages，而不是別的地方？**
  它免費、穩定、你本來就有 GitHub 帳號，而且更新流程單純（上傳檔案就自動更新網站）。

- **為什麼登入 GitHub 時用「權杖」這種方式，而不是一般帳號密碼登入？**
  因為過程中發現，常見的 `gh` 登入指令要求一個叫「read:org」的額外權限，但你給的權杖沒有那個權限，會卡住。我改用「環境變數帶權杖」的方式繞過去（環境變數就是臨時塞給程式看的一個設定值，像遞一張小抄），只需要基本的 repo 權限就能成功，省得你回去重設權杖。

- **為什麼備份的範例檔要設定「不上傳」？**
  備份的用意是留一份安全底稿在你電腦。如果它也上傳到 GitHub，網站上就會出現兩個長很像的檔，反而容易點錯、改錯。所以用 `.gitignore`（一張「這些檔案不要上傳」的名單）把它擋在本地。

## 6. 注意事項與常見問題

- **範例檔（`index_範例檔_…`）請別動。** 它是你的安全底稿，萬一哪天改壞了，靠它就能還原。
- **網站更新沒馬上出現是正常的。** GitHub 需要一點時間，重新整理頁面或等 1 到 2 分鐘就會好。
- **GitHub 上看到的 repo，語言顯示「HTML 100%」是紅色的，那是正常的。** 紅色只是 GitHub 給 HTML 這個語言配的顏色，不是錯誤或警告。
- **倉庫是公開的，請不要在內容裡寫入任何密碼、權杖或個資。** 任何人都看得到。
- **安全提醒（請盡快處理）：** 佈署時用過的 GitHub 個人存取權杖（PAT，就是代替密碼的一串通行碼），曾經出現在對話裡，建議你到 GitHub 的 Token 設定頁（https://github.com/settings/tokens）把它刪掉或重設。刪掉不會影響已經上線的網站。如果之後我還要幫你上傳、剛好憑證失效，再給我一組新的即可。
